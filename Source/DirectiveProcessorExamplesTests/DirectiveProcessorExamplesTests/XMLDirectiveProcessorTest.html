Project:

Delivery Service

Use Case Name:

Login to Account

Use Case Number:

UC001

Use Case Author:

Jim Lavin

Last Revised:

11/10/2003

Summary:

Used to authenticate and identify the user with the web application.

Actors:
  1. Sender
  2. Delivery Clerk
  3. Courier
  4. SuD
  5. DBMS

Preconditions:
  1. The user's information must exist in the DBMS.

Basic Course:

 

Initialization Steps:
  1. The user navigates to the main web page.
  2. The user enters his/her email address in the Email field.The user enters his/her password in the Password field.The user clicks on the Submit icon.The SuD validates that the email address and password have been entered.The SuD queries the User Table for the user credentials.The SuD creates an instance of a User object stores it in the Session State.The SuD displays the message "Welcome, User Name" in place of the login fields.
  3. The SuD updates the user menu with appropriate menu options.

Process Steps:
  1. The user navigates to the main web page.
  2. The user enters his/her email address in the Email field.The user enters his/her password in the Password field.The user clicks on the Submit icon.The SuD validates that the email address and password have been entered.The SuD queries the User Table for the user credentials.The SuD creates an instance of a User object stores it in the Session State.The SuD displays the message "Welcome, User Name" in place of the login fields.
  3. The SuD updates the user menu with appropriate menu options.

Termination Steps:
  1. The user navigates to the main web page.
  2. The user enters his/her email address in the Email field.The user enters his/her password in the Password field.The user clicks on the Submit icon.The SuD validates that the email address and password have been entered.The SuD queries the User Table for the user credentials.The SuD creates an instance of a User object stores it in the Session State.The SuD displays the message "Welcome, User Name" in place of the login fields.
  3. The SuD updates the user menu with appropriate menu options.

Exceptions:
  1. Either the email address or password has not been entered. The SuD redisplays the web page with an error indicating the missing field.
  2. The SuD cannot validate the user credentials. The SuD redirects the user to an error page indicating the information that is incorrect.

Postconditions:
  1. A valid user object has been stored to the Session State.
  2. The welcome message is displayed.
  3. The proper menu is displayed for the user.

Screens Referenced:
  1. Login Page

System Interfaces:
  1. DBMS

DBMS Interfaces:
  1. User Table

USE CASE

Project:

Delivery Service

Use Case Name:

Retrieve Lost Password

Use Case Number:

UC002

Use Case Author:

Jim Lavin

Last Revised:

11/10/2003

Summary:

Process used to recover a lost user password. The password is reset and then sent via email.

Actors:
  1. Sender
  2. Delivery Clerk
  3. Courier
  4. SuD
  5. DBMS

Preconditions:
  1. The user must have an account on the System.

Basic Course:

 

Initialization Steps:
  1. The user has entered an invalid password and has clicked on the link to retrieve a lost password.
  2. The user enters his/her email address and clicks on the submit icon.The SuD looks up the email address.The user clicks on the Submit icon.The SuD generates a new password.The SuD updates the user's record in the User TableThe SuD emails the new password to the user.
  3. The SuD has found the user and sent a new password to the user.

Process Steps:
  1. The user has entered an invalid password and has clicked on the link to retrieve a lost password.
  2. The user enters his/her email address and clicks on the submit icon.The SuD looks up the email address.The user clicks on the Submit icon.The SuD generates a new password.The SuD updates the user's record in the User TableThe SuD emails the new password to the user.
  3. The SuD has found the user and sent a new password to the user.

Termination Steps:
  1. The user has entered an invalid password and has clicked on the link to retrieve a lost password.
  2. The user enters his/her email address and clicks on the submit icon.The SuD looks up the email address.The user clicks on the Submit icon.The SuD generates a new password.The SuD updates the user's record in the User TableThe SuD emails the new password to the user.
  3. The SuD has found the user and sent a new password to the user.

Exceptions:
  1. The user did not enter an email address. The SuD redisplays the web page with an error indicating the field is required.
  2. The SuD could not find the email address entered. The SuD redisplays the web page with an error page indicating the email address was not found.

Postconditions:
  1. The user's password has been set to a temporary password and it has been emailed to the user.

System Interfaces:
  1. SMTP Server
  2. DBMS

DBMS Interfaces:
  1. User Table

Issues:

Assumptions:
  1. All users who access the system will have access to a web browser.
