
Various examples of Directive Processors for T4 customisation.
for Visual Studio 2012 RTM


See ReadMe.txt in each project for details of source.

Build Project, and ensure VSIX is set to Start Up Project.
VSIX project is set up for debugging via the Experimental Instance.

Right-Click VSIX project and select Debug. The Experimental Instance will open.
You can use Tools > Extensions to see if the VSIX has loaded.
The VSIX options of the project properties have been set to build and deploy there.

In the Experimental Instance that opens, open the test solution that is part of this distribution, 
and see the operation of the various Directive Processors by running "Run Custom Tool" for the associated .tt
